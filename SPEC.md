# Chronos — Product Specification Document

## 1. System Overview
Chronos consists of two layers:
1. **Orchestration layer** — IBM Bob custom modes running against a target repo, producing structured JSON/Markdown artifacts.
2. **Presentation layer** — a static/lightweight dashboard that reads those artifacts and renders them visually.

```
chronos/
├── AGENTS.md                  # generated by Bob /init
├── .bob/
│   └── custom_modes.yaml      # Cartographer, Modernizer, Compliance Officer
├── legacy-repo/                # sample target codebase (Java 11)
│   └── ...
├── outputs/
│   ├── dependency-map.json
│   ├── changelog.md
│   └── compliance-report.json
├── dashboard/                  # simple viewer app
│   └── ...
└── README.md
```

## 2. Bob Setup Sequence
1. Open `chronos/` as workspace root in Bob IDE.
2. Run `/init` in Code mode → generates root `AGENTS.md`.
3. Set auto-approval to **Read only** initially.
4. Create the three custom modes (Section 3).
5. Run modes in sequence: Cartographer → Modernizer → Compliance Officer.
6. Collect outputs into `outputs/`.
7. Build dashboard reading from `outputs/`.

## 3. Agent Mode Specifications

### 3.1 Cartographer
- **Slug:** `cartographer`
- **Tool access:** Read-only (file read, search, list — no write/execute)
- **Role definition:**
  > You are a static analysis agent. Your job is to read the target codebase, identify all files, classes, functions, and their dependencies (imports, calls, inheritance), and produce a structured map of the system. You do not modify any files.
- **Task prompt (example):**
  > Scan the `legacy-repo/` directory. Identify every Java file, its package, its imports, and which internal classes/methods it calls. Output a JSON file at `outputs/dependency-map.json` with this structure: `{ "files": [{ "path": "", "package": "", "imports": [], "calls": [], "deprecated_apis_used": [] }] }`. Also produce a short human-readable architecture summary in `outputs/architecture-summary.md`.
- **Output contract:** `outputs/dependency-map.json`, `outputs/architecture-summary.md`

### 3.2 Modernizer
- **Slug:** `modernizer`
- **Tool access:** Read + write + edit (no delete of unrelated files)
- **Role definition:**
  > You are a code modernization agent. Using the dependency map as context, refactor deprecated or outdated patterns in the codebase to current standards (e.g., upgrading Java 11 syntax/APIs to Java 21+/25 equivalents) while preserving existing logic and behavior exactly. Do not change business logic. Log every change you make.
- **Task prompt (example):**
  > Using `outputs/dependency-map.json` as context, refactor `legacy-repo/` to replace deprecated APIs and outdated patterns flagged in `deprecated_apis_used`. Preserve method signatures and behavior. For every change, append an entry to `outputs/changelog.md` with: file, before snippet, after snippet, and a one-line reason.
- **Output contract:** modified files in `legacy-repo/`, `outputs/changelog.md`

### 3.3 Compliance Officer
- **Slug:** `compliance-officer`
- **Tool access:** Read + limited write (only to `outputs/`, not to source files)
- **Role definition:**
  > You are a compliance and security auditor. Scan the modernized codebase against a defined rule set (see Section 4) covering GDPR, HIPAA, and general secure-coding practices. You do not modify source code — you only flag issues with severity, location, and the specific rule violated.
- **Task prompt (example):**
  > Scan `legacy-repo/` against the rules in Section 4 of `SPEC.md`. For each violation found, record: file, line, rule ID, severity (High/Medium/Low), description, and suggested fix. Output `outputs/compliance-report.json` with structure: `{ "findings": [{ "file": "", "line": 0, "rule_id": "", "severity": "", "description": "", "suggested_fix": "" }], "summary": { "high": 0, "medium": 0, "low": 0 } }`.
- **Output contract:** `outputs/compliance-report.json`

## 4. Compliance Rule Set (MVP — keep small and real)
Choose depth over breadth. Suggested starter set, each tied to a real, named standard:

| Rule ID | Standard | Check |
|---|---|---|
| SEC-001 | General secure coding | Hardcoded credentials, API keys, or secrets in source |
| GDPR-032 | GDPR Art. 32 | Personal data (name, email, SSN-like fields) stored or transmitted without encryption |
| GDPR-005 | GDPR Art. 5 | No apparent data-minimization — logging full personal records unnecessarily |
| HIPAA-164 | HIPAA §164.312 | PHI-like fields (medical, health) not access-controlled or encrypted |
| SEC-002 | OWASP | Use of deprecated/insecure crypto or hashing (e.g., MD5, DES) |

Seed the sample repo with 2–3 of these deliberately so the Compliance Officer has real things to find.

## 5. Sample Legacy Repo Requirements
- Language: Java 11
- Size: small — 4–8 files, enough to show real structure without overwhelming a demo
- Must include:
  - At least 2 deprecated API usages (Modernizer target)
  - At least 1 hardcoded secret (Compliance target)
  - At least 1 unencrypted personal/health data field (Compliance target)
  - A simple internal dependency chain (2–3 classes calling each other) so the Cartographer's map is non-trivial

## 6. Dashboard Specification
- **Purpose:** visualize outputs, not recompute anything — pure display layer.
- **Tech:** single HTML/JS page (or React) — no backend required; reads local JSON/MD files.
- **Views:**
  1. **Dependency Map** — node/edge graph or simple tree view of `dependency-map.json`
  2. **Before/After Diff** — side-by-side or unified diff view sourced from `changelog.md`
  3. **Compliance Report** — table of findings from `compliance-report.json`, color-coded by severity
- **Non-goals for MVP:** live re-running of agents from the dashboard, editing capability, auth.

## 7. Data Contracts (Schemas)

### dependency-map.json
```json
{
  "files": [
    {
      "path": "string",
      "package": "string",
      "imports": ["string"],
      "calls": ["string"],
      "deprecated_apis_used": ["string"]
    }
  ]
}
```

### changelog.md (format, not JSON)
```
## <file path>
**Before:**
```<snippet>```
**After:**
```<snippet>```
**Reason:** <one line>
```

### compliance-report.json
```json
{
  "findings": [
    {
      "file": "string",
      "line": 0,
      "rule_id": "string",
      "severity": "High | Medium | Low",
      "description": "string",
      "suggested_fix": "string"
    }
  ],
  "summary": { "high": 0, "medium": 0, "low": 0 }
}
```

## 8. Demo Flow (for the 3-minute video)
1. Show the raw legacy repo (10s)
2. Run Cartographer → show generated dependency map / diagram (30s)
3. Run Modernizer → show before/after diff on 1–2 concrete examples (45s)
4. Run Compliance Officer → show flagged findings on the dashboard (45s)
5. Show final dashboard view, summarize impact (30s)
